Deepfakes

如何使用 Deepfakes 换脸

一、准备工作

  1. 运行环境

    • python3.6 pip3.6
    • macOS需要安装: xquartz(手动安装)
    • 如果你有N卡(GPU训练更快)，Nvidia graphics card：You should install CUDA (https://developer.nvidia.com/cuda-zone) and CUDNN (https://developer.nvidia.com/cudnn). If you do not plan to build Tensorflow yourself, make sure you install no higher than version 9.0 of CUDA and 7.0.x of CUDNN
    • docker，通过DockerFile构建，环境与官方保持一致
    • docker中使用到了 jupyter notebook ,可以根据终端提示的链接打开网页，可上传文件、运行python代码

  2. 获取deepfakes工具包

    $ git clone https://github.com/deepfakes/faceswap.git

  3. 补全依赖

打开工具包目录下的requirements.txt 补全所有依赖 requirements.txt 为了安装快速,可以使用国内源，示例代码如下:

# 逐个安装
$ sudo pip install pynvx -i https://mirrors.aliyun.com/pypi/simple/
# 批量安装
$ sudo pip --no-cache-dir install -r requirements.txt
  1. 安装完所有依赖运行 setup
$ python setup.py
  1. docker 配置
$ docker build -t deepfakes-cpu -f Dockerfile.cpu .
# Nvidia-Docker is only supported on Linux.Only CPU is supported in Docker for your system
# 如果是Mac系统或者不支持GPU
$ docker run -p 8888:8888  --hostname deepfakes-cpu --name deepfakes-cpu  -v /Users/plucky/plucky/github/python/faceswap:/srv  deepfakes-cpu
# 如果是Linux系统支持GPU
xhost +local:
nvidia-docker run -p 8888:8888 \
            --hostname deepfakes-cpu --name deepfakes-cpu \
            -v /Users/plucky/plucky/github/python/faceswap:/srv \
            -v /tmp/.X11-unix:/tmp/.X11-unix \
            -e DISPLAY=unix$DISPLAY \
            -e AUDIO_GID=`getent group audio | cut -d: -f3` \
            -e VIDEO_GID=`getent group video | cut -d: -f3` \
            -e GID=`id -g` \
            -e UID=`id -u` \
            deepfakes-cpu
# 执行以上代码后会提示
The Jupyter Notebook is running at:
http://(deepfakes-cpu or 127.0.0.1):8888/?token=your token
# 新开一个 Terminal 执行 `docker exec -it deepfakes-cpu bash` 进入bash
# 打开提示的链接，会要求输入token,提示输入 `jupyter notebook list`
# 在新开的 Terminal 执行 `jupyter notebook list`，可以看到 token,在网站 输入token 就可以打开网站首页了

# 因为创建过 docker 容器，所以之后只需要启动容器,然后在进入容器的bash运行代码即可
$ docker start -i xxxx
$ docker exec -it deepfakes-cpu bash

二、重要信息

明确一点 接下来所有的工作都是在 docker 容器中进行,本来以为下载源码就可以直接运行 faceswap 但一直出错；后来发现其实创建 docker 的目的是在 docker 中运行 faceswap . Dockerfile.cpu 或 Dockerfile.gpu 已经为我们配置好了运行 faceswap 的 Linux 环境。启动 docker 容器的时候，将我们的源码挂载到了 Linux 中 srv 目录下，我们运行代码时需要 cd ../srv/ 因为默认打开终端是 notebook目录下。

三、检测环境

如果执行 python faceswap.py -h 无报错，那么恭喜你环境已经配置成功了.(注意：GUI模式 由于设备原因，在本例中暂不支持)

四、准备素材

由于机器需要学会识别出两个需要处理的脸，所以，需要交换的两张脸都需要图片素材以供机器学习，越多越好

  1. 从网上下载图片
  2. 或从视频提取图片：ffmpeg -i /path/to/my/video.mp4 /path/to/output/video-frame-%d.png

五、提炼人脸 EXTRACT

# To convert trump:
$ python faceswap.py extract -i ~/faceswap/photo/trump -o ~/faceswap/data/trump
# To convert cage:
$ python faceswap.py extract -i ~/faceswap/photo/cage -o ~/faceswap/data/cage
# My Work
$ python faceswap.py extract -i _data/in/lyf/ -o _data/out/lyf/ -A dlib
$ python faceswap.py extract -i _data/in/video/frames_in/ -o _data/out/frames/ -A dlib

如果出现错误 ValueError: Error initializing Detector

如果有GPU 使用 dlib 即可 '-D dlib-cnn'

$ python faceswap.py extract -i /input/path -o /output/path -D dlib-cnn -A dlib

六、训练模型 TRAIN

-A 表示被换的脸，源脸 -B 表示需要换成的脸，目标脸 -m 输出训练模型; 如果电脑配置不够(Docker配置的内存及CPU可按需调整)，建议使用 -t LowMem (否则卡到怀疑人生)

$ python faceswap.py train -A ~/faceswap/data/trump -B ~/faceswap/data/cage -m ~/faceswap/models/
# or -p to show a preview
$ python faceswap.py train -A ~/faceswap/data/trump -B ~/faceswap/data/cage -m ~/faceswap/models/ -p
# My Work
$ python faceswap.py train -A _data/out/frames/  -B _data/out/lyf  -m _data/out/model -t LowMem

谨记：1. 训练过程中，按 Enter 结束训练，保存模型需要一丢丢时间，如果强制退出可能会损坏模型 2. 在停止结束后，备份好你的模型文件，因为当你再次开始时，有可能损毁你的模型

七、换脸 CONVERT

-i 需要被换脸的原始图片 -o 换脸后的输出文件夹 -m 模型文件目录

$ python faceswap.py convert -i ~/faceswap/photo/trump/ -o ~/faceswap/output/ -m ~/faceswap/models/
# my work
$ python faceswap.py convert -i _data/in/video/frames_in/  -o  _data/in/video/frames_out/  -m  _data/out/model/ -t LowMem

八、将图片转换成视频

$ ffmpeg -i video-frame-%0d.png -c:v libx264 -vf "fps=25,format=yuv420p" out.mp4
